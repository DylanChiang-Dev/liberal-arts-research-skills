## Claude Code plugin

來源：
- Claude Code 官方文件：`https://docs.claude.com/en/docs/claude-code/plugins`

核實結果：
- plugin manifest 路徑：`.claude-plugin/plugin.json`。
- `plugin.json` 唯一必填欄位：`name`。
- 官方支援的 metadata 欄位包括：`description`、`version`、`author`、`homepage`。
- 官方支援的 content 欄位包括：`commands`、`agents`、`skills`、`hooks`、`mcpServers`、`settings`。
- `skills` 可指定 plugin 內技能目錄，例如 `"skills": "./skills"`。
- marketplace 需要倉庫根目錄 `marketplace.json`，包含 `name`、`owner`、`plugins[]`；plugin entry 包含 `name`、`repo`、`description`、`version`、`author`。

待確認：
- 若只在本倉庫放 `.claude-plugin/plugin.json` 而不新增 marketplace，是否可直接從本地路徑安裝；本輪只做 repo 內 manifest，不新增 marketplace。

## Codex plugin / skills

來源：
- Codex manual：`https://developers.openai.com/codex/plugins/build`
- Codex manual：`https://developers.openai.com/codex/skills`
- 本機 Codex `plugin-creator` skill reference：`/Users/dc/.codex/skills/.system/plugin-creator/references/plugin-json-spec.md`

核實結果：
- Codex plugin manifest 路徑：`.codex-plugin/plugin.json`。
- 最小 plugin manifest 可包含：`name`、`version`、`description`、`skills`。
- `name` 使用穩定 kebab-case，Codex 以它作為 plugin identifier / component namespace。
- `skills` 可指定 plugin 內技能目錄，例如 `"skills": "./skills/"`。
- Codex 也可直接讀取 repo/user/admin/system skills；repo/user 常用位置包括 `$REPO_ROOT/.agents/skills` 與 `$HOME/.agents/skills`。
- local plugin 分發通常透過 marketplace：repo marketplace 可放 `$REPO_ROOT/.agents/plugins/marketplace.json`，personal marketplace 可放 `~/.agents/plugins/marketplace.json`。
- marketplace plugin entry 必含：`name`、`source`、`policy.installation`、`policy.authentication`、`category`。
