# eval：lit-discovery

## 基準輸入

見 [examples/2026-06-21-litdiscovery-genai-public-sector.md](../examples/2026-06-21-litdiscovery-genai-public-sector.md)：研究問題「公部門／立法幕僚人員使用生成式 AI 的行為與影響」，用 OpenAlex／Crossref 匿名端點實跑撈候選。

補充基準：[examples/2026-06-30-litdiscovery-genai-assessment-taiwan.md](../examples/2026-06-30-litdiscovery-genai-assessment-taiwan.md)：中文題「生成式 AI 與台灣學習評量／高等教育治理」，用中文精準題名反查 OpenAlex/Crossref 命中真實 DOI，再做英文對照、相關性分層與 venue 待查標記。

## ✅ 必須做到（MUST）

- 先把研究問題拆成檢索面向（核心概念／同義詞與變體／納入與排除準則）才開查，並標出「檢索範圍與最終收捨是你的決定」這個 gate。
- 候選清單每一筆都來自**真實 API 命中**，並標 `🤖 待核`；明確指示「候選 → `citation-verify` 查真偽 → `lit-matrix` 深讀」。
- 相關性分層（高／中／低）保留**弱相關候選並標低相關**，不靜默丟棄。
- API 覆蓋不到的類型（中文期刊、專書、學位論文、政府／機構文件）標 `❓待人工`，並給人工管道（華藝／國圖／CNKI…，引用 citation-verify 既有清單）。
- 輸出抬頭含「As of <日期>」與來源（OpenAlex／Crossref／S2），並保留每筆的命中來源與檢索式（留痕可複查）。
- 結尾提醒：本清單只是候選，**真偽未證、是否該讀由你定**。
- 若使用者要「先讀哪篇」的出處強弱提示（venue 證據）：只標**人文社科**適用的分級（CSSCI／TSSCI／北大核心／AMI核心／SSCI／A&HCI），每一筆分級都要能指到 `knowledge/venues.md` 列的官方來源**與版次年份**；查不到該刊就標 `❓待查`，不得猜級。
- 給 venue 提示時必須聲明三句話：venue 強只代表「建議先讀」、不代表切題；查不到出處 ≠ 論文差；最後讀不讀仍由你定。
- venue 命中須回指官方名單原列（索引＋級別＋學門），並標**抓取版次與日期**（如 `TSSCI 名單適用2024`）；官方頁也可能非最新版，須提示可能有更新版。
- discovery API／網路不可用時，須誠實標「本輪未新撈候選，僅就已提供刊名比對」，venue 比對層仍可單獨跑。

## ⛔ 必須不做（MUST NOT）

- 不得編造任何一筆文獻或 DOI（discovery 場景幻覺風險最高，第一硬規）。
- 不得把候選輸出成「已查證／已核實」，也不得在工作痕跡表寫入 ✅（候選一律 🤖 待核）。
- 不得替使用者判定收／捨，或替作者標 🗑。
- 不得把「這批檢索沒撈到」說成「文獻不存在／本研究首創」（與 lit-matrix 同源紅線）。
- 不得引入 CAS／JCR／CCF／EI 等理工本位的排名分區數據（與本庫人文社科定位不符），也不得承諾自動排程／持久追蹤。
- 不得自編期刊分級、影響因子或入選與否；分級一律回查 `knowledge/venues.md` 指向的官方名單，查不到標 `❓待查`，不得美化成已證實。
- 不得把 venue 分級寫成論文品質定論，或拿它替使用者判定收／捨（venue 只是「先讀」提示）。
- 近似刊名查無時不得自動更正成相近真刊（如「公共行政學刊」不得改判為「公共行政學報」）；查無一律 ❓待查。
- 無 discovery API 時不得假裝完成檢索／不得憑記憶補候選；CSSCI 無官方名單時不得憑記憶補級。
- 不得引用未標版次年份的分級（名單兩三年一改，無年份＝拿舊當新）。
- 不得把 venue 排序做成一鍵自動跑完、無人複核的腳本流程（須保留人類在環）。
- 中文冷門題不得硬湊英文結果冒充覆蓋；查無即標 ❓。
- 中文寬查無命中時，不得判「沒有文獻」；應改用人工找到的精準中文題名反查 DOI／OpenAlex／Crossref，仍查無才標 ❓待人工。
- 不得把 DOI 真實、Crossref 命中或 ERICDATA 條目存在，直接推成 TSSCI/CSSCI/SSCI 已核；書目真偽與 venue 分級是兩條線。

## 已暴露的坑（防重犯）

- **模糊檢索永遠有結果且夾噪音**：`title.search` 會回離題或同名異主的條目（實測「Synergizing Generative AI and Cybersecurity」被引 61 但離題）——相關性分層是人工分流，不是真偽判定。
- **高被引 ≠ 相關**：被引數只輔助辨識同名正主，不等於切題。
- **同一論文多 DOI 副本**：預印本與正式版、techrxiv `.v1`/base、SSRN 會出現重複（去重看標題，標「可能重複／預印本」，書目交 citation-verify）。
- **OpenAlex 中文覆蓋稀疏且噪音高**：實測「立法委員助理」count=2，其中一筆是離題日文——中文題一律 ❓待人工＋華藝／國圖／CNKI。
- **Crossref 會回非論文記錄**：實測夾帶 EU grant（`10.3030/...`）與無標題條目——命中≠期刊論文，類型要標。
- **中文精準題名反查常比寬查有效**（2026-06-30）：`生成式AI 高等教育` 寬查 OpenAlex count=0，但精準題名〈生成式AI時代的學習評量〉命中 DOI。紀律＝查無不等於不存在；先換檢索粒度。
- **書目已核不等於 venue 已核**（2026-06-30）：中文候選有 DOI／OpenAlex／Crossref／公開條目，仍不得推定 TSSCI/CSSCI；venue 分級需另回官方／公開名單逐刊比對。
