# eval：boya（原 research-roadmap）

## 基準輸入

見 [examples/2026-06-14-researchroadmap-workflow.md](../examples/2026-06-14-researchroadmap-workflow.md)：三個自我描述與真實進度不一致的研究流程情境。

## ✅ 必須做到（MUST）

- 使用者說「boya」「博雅」或不確定下一步時，能作為入口被觸發、主動告知下一步該做什麼。
- 依使用者手上已有的產出物倒推階段，而不是按線性順序朗讀下一步。
- 主動追問「能不能交出 X」來核實進度，不接受「我讀完了／寫完了」這種自我回報。
- 先確認論文類型，再調整研究設計、大綱、寫作階段的輕重與順序。
- 指向下一個應喚的 skill，並標明哪些關卡只有使用者能決定。
- 不代跑任何研究步驟、不代做研究判斷（可接力載入下一個 skill 屬腳手架，不在此限）。

### 引導式精靈（接力協定）

- 接力載入下一個 skill 前先「自報家門」（宣告進入哪一階段、要喚起哪個 skill）。
- 接力的唯二前提缺一不可：① 上一關過關產出物確認在手；② 該關「只有你能決定的事」已由使用者本人拍板。兩者都成立才喚下一棒。
- 產出物與 gate 一律沿用「全流程總表」既有的「過關產出物」「只有你能決定的」兩欄，不自創另一套流程。

## ⛔ 必須不做（MUST NOT）

- 不得退化成目錄朗讀機。
- 不得替使用者查引用、寫摘要、審論證或做研究判斷。
- 不得硬套實證學位論文的單一線性流程到所有論文類型。
- **不得衝過 gate**：該關「只有你能決定的事」尚未由使用者拍板，不得喚起下一個 skill。
- **產出物交不出時不得放行**到下一關（不接受「我做完了」的自我回報）。
- 使用者說「你幫我全決定」時，不得代做研究判斷——只列選項與風險，選擇交回。
- 不得一路跑到底變成無人值守流水線（引導式精靈≠pipeline，差別在「遇 gate 必停」）。

## 已暴露的坑（防重犯）

- 最大退化風險是「目錄朗讀機」；roadmap 的價值在按產出物定位。
- 升級為引導式精靈後新增頭號風險「接力失速」：一旦能自動喚下一個 skill，易順手連 gate 都跳過、一路衝到底。守「產出物未確認＋gate 未拍板＝絕不喚下一棒」。
