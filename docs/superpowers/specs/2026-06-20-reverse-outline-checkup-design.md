# 設計：outline-builder「逆向體檢」模式（reverse-outline check-up）

> 日期：2026-06-20
> 範圍：合一份 spec，落在 `outline-builder`。涵蓋兩個借來的想法——#1 逆向大綱（整篇／章層級）、#2 段落微觀工法（段落層級）——它們是同一招的兩個放大倍率，故合併。

## 背景

參考 `Master-cai/Research-Paper-Writing-Skills`（將彭思达論文寫作筆記打包成的 ML/CV/NLP 寫作 skill），**只借想法、不抄任何內容，全部自行重寫**。

該專案是理工科導向，teaser 圖、pipeline 圖、實驗章節、minimal-ink 表格等 STEM 工法**不適用**於人文社科。能轉移的只有**與學科無關的「文字邏輯工法」**那一層，具體是兩個想法：

1. **逆向大綱（reverse outlining）當體檢工具**：拿已寫好的草稿反推骨架——抽出每段主題句與證據，檢查「主題句扣不扣得回主線、證據扣不扣得回主題句」，扣不上的就砍／補／搬。
2. **段落微觀工法**：一段一意、首句點題、句與句關係只有四種（因果／轉折／推論／細化）、名詞自足且術語先定義再用。

這兩者在來源專案本就綁在一起教：**逆向抽骨架這個動作，正是用來檢查段落是否一段一意、是否扣得回主線**。#1 是整篇／章倍率、#2 是段落倍率，同一個體檢動作。

## 問題

`outline-builder` 目前是「**從材料正向搭骨架**」（寫之前用）：盤點材料 → 選結構模式 → 長章節大綱 → 段落 claim–evidence–warrant → 檢查章節邏輯。它**已有半套逆向能力**：

- 第 3 步「每章一句話串起來讀」＝章層級逆向大綱雛形。
- 第 5 步「抽掉這章影響嗎」＝砍鬆散。
- 陷阱 5「完整性幻覺」叫人用第 3 步去測。

**缺的是**：把逆向體檢**往下延伸到段落層級**，而且**對「已經寫好的草稿」做**。現況沒有「拿成稿反推、逐段體檢」這個動作；段落微觀工法（#2）也沒有成文的檢查清單。

## 目標

給 `outline-builder` 新增一個**「逆向體檢」模式**：拿使用者已寫好的草稿，反推每段骨架 → 用微觀清單逐段檢查 → 標出扣不回主線、該砍／該補／該搬的地方。**不新增 skill**、**不碰其餘 skill 內臟（除交叉指引）**、**不代筆**。

## 非目標（明確不做）

- 不新增獨立 skill；逆向體檢是 `outline-builder` 的一個模式。
- 不搬 STEM 專屬工法（teaser／pipeline 圖、實驗章節、minimal-ink 表格、「別像在打補丁的 baseline」）。
- 不替作者改寫句子（改寫屬 `style-tune`，且守不代寫紅線）。本模式只診斷、給方向。
- 不重新定義 warrant（已在第 4 步定義，逆向體檢引用之）。
- 不與 `self-review` 的「一桌審稿人」整篇審稿重複——後者是宏觀論證批判，本模式是段落層級的邏輯／流暢體檢，互相交叉指引。

## 設計

### ① 它是什麼、何時用

- **正向模式（既有）**：從材料搭骨架，寫之前用。
- **逆向體檢模式（新增）**：拿已寫好的草稿反推骨架、抓鬆散，寫之後用。
- 觸發語（寫進 description）：「幫我看這稿讀不讀得順」「邏輯有沒有洞」「這幾段鬆不鬆散」「幫我檢查我寫好的這幾段」。

### ② 工作流：三個動作

**動作 1　反推骨架**
拿草稿，逐段抽出「這段的主題句（一句話它想說什麼）＋它給的證據」。先章層級（沿用既有第 3 步「每章一句話串起來讀」），再往下到段落層級逐段抽。

**動作 2　逐段套微觀清單（＝想法 #2）**
對每一段檢查四項：

| 檢查項 | 問法 |
|---|---|
| 一段一意 | 這段只講一件事嗎？混了兩三件就標出。 |
| 首句點題 | 第一句就講了那件事嗎？沒有 → 指出該把哪句提前。 |
| 句關係清楚 | 每一句到下一句是 **因果／轉折／推論／細化** 哪一種？接不上＝邏輯斷。 |
| 名詞／術語 | 名詞能自己站住嗎？術語先定義再用了嗎？ |

**動作 3　對主線**
逐層扣回：每段主題句扣得回該章論點嗎？章論點扣得回研究問題嗎？證據扣得回主題句嗎？扣不上的 → 標處置：**該砍／該補／該搬**。

### ③ 輸出：逆向體檢表

```markdown
# 逆向體檢報告：<稿件／章節>

| 段落 | 抽出的主題句 | 一段一意 | 首句點題 | 句關係 | 扣回主線 | 處置 |
|---|---|---|---|---|---|---|
| 第2章第3段 | （它其實在說……） | ✓ | ✗ 首句沒點題 | ✓ | ✓ | 改：把第3句提到開頭 |
| 第2章第4段 | （說不清） | ✗ 混了兩件事 | — | ✗ 第2句突兀 | ✗ 扣不回 | 拆成兩段或刪 |

## 一句話總評
<最該先處理的鬆散點>
```

### ④ 邊界（與 Boya 立場一致）

- **只診斷、不代筆**：標出洞與方向，不替作者改寫句子。需要改寫 → 交給 `style-tune`。
- **骨架是作者的**（沿用既有鐵律 1）：補不補、怎麼補、砍不砍，作者決定。
- **DRY 分工**：
  - warrant 不重新定義，指回第 4 步。
  - `style-tune`（聲音／AI 腔）與 `self-review`（整篇審稿）各加一句交叉指引：「讀起來鬆散／邏輯斷 → 回 outline-builder 跑逆向體檢，再來潤／審」。

## 具體改動清單

1. `skills/outline-builder/SKILL.md`：
   - description 補逆向體檢的觸發語。
   - 新增「逆向體檢模式」工作流（動作 1–3）與輸出表；於開頭區分正向／逆向兩模式。
   - 微觀清單（#2）寫成檢查項；陷阱區補逆向體檢易犯錯（如「只抽骨架不對主線」「把改寫當診斷」）。
2. `skills/style-tune/SKILL.md`：邊界情況／串用處加一句交叉指引（鬆散先回 outline-builder 逆向體檢）。
3. `skills/self-review/SKILL.md`：邊界情況／串用處加一句交叉指引（段落層級鬆散 → outline-builder 逆向體檢）。
4. `evals/outline-builder.md`：補逆向體檢回歸斷言。
5. `examples/2026-06-20-outline-reverse-checkup-case.md`：一份實跑真錄。
6. `VERIFICATION.md`：補實測列。

## 驗收（依 CONVENTIONS §4）

- `outline-builder` 升版號前，需有 ≥1 篇 `examples/` 實跑真錄（建議重用既有碩論章節材料），展示逆向體檢表被正確產出。
- 回歸斷言寫進 `evals/outline-builder.md`，至少含：
  - 逆向體檢必須「反推骨架 → 微觀清單 → 對主線」三動作齊全，不可只抽骨架不對主線。
  - 只診斷不代筆（不得替作者改寫整段）。
  - 微觀清單四項逐段套用。
- 每次實測後 `VERIFICATION.md` 補一行。
