# 設計：research-roadmap「引導式精靈」模式（guided dispatcher）

> 註（2026-06-23）：`research-roadmap` 已更名 `boya`；本設計文件為歷史記錄，內文沿用舊名。

> 日期：2026-06-20
> 範圍：一份 spec，主要落在 `skills/research-roadmap/SKILL.md`。把 research-roadmap 從「只指路、要使用者自己一個一個手動喚 skill」升級成「喚一次就被一步一步帶著走、自動接力到下一個 skill，但每個『只有你能決定』的關卡硬停等人」。**不新增 skill、不碰其餘 12 個 skill 的核心工作流（至多加一句回報交叉指引）、不做無人值守 pipeline。**

## 背景

參考 superpowers（obra/Jesse Vincent）的 skill 接力機制。它讓「打開一個就一路走完」靠三個寫在 SKILL.md 裡的 prose 機制，而非程式碼編排：

1. **自報家門**：每個 skill 開頭宣告「我正在使用 X skill」。
2. **硬跳轉指令**：skill 在交接點明寫「完成後改用 Y skill」，agent 讀到就用 Skill 工具把下一個叫起來。
3. **checkpoint**：關鍵環節停下來等人確認再往下。

superpowers 的「一路走」實際上是**「自動接力 ＋ 每關停下等人」**，不是無人值守跑完。這一點正是它能和 Boya 相容的關鍵。

## 問題

`research-roadmap` 目前是**指路牌、不是會接力的協調者**。它的 description、角色說明、鐵律 1 三處明寫「只導航、不代跑」「不是按一個鍵就自動產出論文」「人工帶路、不是自動跑完的流水線」。具體行為：

- 它會說「下一步喚 `topic-refine`」，但**不會自動喚起** `topic-refine`。
- 使用者得**自己一個一個手動喚** 13 個 skill，還要自己記得把上一關的產出物餵給下一關。

痛點：缺少 superpowers 那種「喚一次就被帶著一步一個 skill 走完整篇」的順手感。使用者問的就是這個：**「開第一個 skill 就像工作流一樣一步步引導我完成整篇，一步一個 skill。」**

同時，Boya 有一條真實紅線不能破：`VERIFICATION.md` 把「多 agent 長跑 orchestrator」「無人值守研究 pipeline」明列為**刻意不採用的重型設計**；整套哲學是「AI 是副駕駛，不是機長」。所以不能做成「AI 一路跑到底、替你把研究判斷全做完」。

## 目標

讓使用者**喚一次 `research-roadmap`，就被一步一個 skill 帶著走完整篇文章**——roadmap 自動接力載入下一個 skill、自動做產出物交接——但**每到「只有你能決定」的 gate 就硬停、把方向盤交回、等使用者拍板，才接力載入下一個 skill**。

一句話：把 research-roadmap 從**指路牌**升級成**會接力的嚮導**，接力的是「流程腳手架」，**不接力的是研究判斷**。

## 非目標（明確不做）

- **不做無人值守 pipeline**：不一路跑到底、不跳過任何 gate、不替使用者做研究判斷（守 `VERIFICATION.md` 既有立場）。
- **不新增 skill**：這是 research-roadmap 的能力升級，不是新 skill；README 計數仍 13。
- **不碰 12 個下游 skill 的核心工作流**：它們各自的「只有你能決定的關卡」「鐵律」「輸出格式」一律不動；至多在尾巴加**一句**回報交叉指引（且此項列為設計選項，預設不加，見設計⑤）。
- **不做程式碼編排／多 agent 長跑**：接力純靠 SKILL.md prose 驅動（同 superpowers），不引入 orchestrator、manifest 分片、子代理長跑。
- **不替使用者把「查無」當「完成」**：產出物核實由使用者回報、roadmap 追問，不自動判定過關。

## 設計

### ① 靈魂區分：引導式精靈 ≠ 無人值守 pipeline

整份設計成立的前提是這張區分表。它界定「採用什麼、仍禁止什麼」，必須寫進 SKILL.md 與 VERIFICATION.md，否則會被誤讀成 Boya 自打嘴巴。

| 面向 | 引導式精靈（**本案採用**） | 無人值守 pipeline（**仍禁止**） |
|---|---|---|
| 接力下一個 skill | ✅ 自動喚起 | ✅ 自動喚起 |
| 「只有你能決定」的 gate | **硬停，等使用者拍板才放行** | 不停，AI 直接代做 |
| 研究判斷（選題／選方法／詮釋／論證主線／署名） | 全留給使用者 | AI 代做 |
| 過關產出物核實 | 使用者回報 ＋ roadmap 追問「能不能交出 X」 | AI 自動判定過關 |
| 失速時 | 停在 gate 等人 | 一路衝到底 |

差別只在一件事：**遇到 gate 會不會停**。會停 → 精靈；不停 → pipeline。

### ② 三個零件

**零件 A — dispatcher 化的 roadmap（接力協定）**
roadmap 在「某一關的 gate 被使用者清掉、且過關產出物確認交出」後，**主動用 Skill 工具喚起下一階段對應的 skill**（對應「全流程總表」的「對應 skill」欄）。喚起前先**自報家門**：「你已過階段 N，我現在喚起 `<skill>` 幫你做階段 N+1 的苦工。」

**零件 B — gate 停點協定（最重要）**
定義「停在哪、怎麼問、收到什麼才放行」：

- **停點來源**：直接用「全流程總表」既有的「**只有你能決定的**」欄與各 skill 既有的同名關卡——不另立一套。
- **停法**：到達 gate 時，roadmap **不喚下一個 skill**，改成把這一關「只有你能決定的事」攤開，問使用者「這一關該你拍板的是 X，你的決定是？」
- **放行條件**：使用者給出**他自己的**判斷（不是 AI 替他選的）後才放行。使用者說「你幫我決定」時，roadmap 守鐵律 5——指出選項與風險，但把選擇交回，不代選。

**零件 C — 產出物交接協定**
上一關的「過關產出物」（總表既有欄位）→ 下一關 skill 的輸入。roadmap 接力時，**先確認上一關產出物真的在手**（問「能不能交出 X」，不接受「我做完了」的自我回報，沿用既有鐵律 1／工作流第 1 步），再把它作為下一個 skill 的起點。

### ③ 自報家門 ＋ 回報核實循環（借 superpowers，落在 roadmap 端）

每一步的循環固定為：

```
roadmap 自報家門：「進入階段 N，喚起 <skill>」
   → 喚起 <skill>，使用者與該 skill 完成這一關的苦工
   → 回到 roadmap：roadmap 問「這一關的過關產出物 X 交得出來嗎？」
       交不出 → 留在本關補（不放行）
       交得出 → 到本關 gate：「只有你能決定的是 Y，你怎麼定？」
           使用者拍板 → 自報家門進入階段 N+1（回到頂端）
```

這個循環把 superpowers 的「自報家門＋硬跳轉＋checkpoint」三件事，全部收斂在 roadmap 一個檔裡實作，**下游 12 個 skill 不需要知道自己被接力**。

### ④ 為什麼接力邏輯集中在 roadmap（架構取捨）

有兩種接法，本案選 A：

- **A · roadmap 中心制（採用）**：所有接力／停點／交接邏輯集中在 `research-roadmap` 一個檔。子 skill 跑完，使用者**回到 roadmap**（整套 skill 的 spine），由 roadmap 核實產出物、守 gate、再喚下一個。**12 個 skill 完全不動。**
  - 優點：最 DRY、最小改動、最貼 Boya「roadmap 是 spine」的既有定位（SKILL.md 末行已自陳「它是整套 skill 的 spine…只串起其餘十一個 skill」）。哲學上也最順——使用者本來就該在每個路口回到 spine 做判斷。
  - 缺點：使用者跑完一個 skill 後要「回到 roadmap」這個動作，不是 skill 自己把人送回。

- **B · skill 尾巴硬跳轉**：每個 skill 尾巴加「完成後 STOP，回報 research-roadmap」。更像 superpowers 的硬跳轉、銜接更無縫。
  - 缺點：碰 12 個檔、有重複（即使每檔只一句）、且讓子 skill 依賴 roadmap，違反「skill 各自獨立可單用」的現況。

**結論**：採 A。B 的「一句尾巴指引」列為**可選增強**（設計⑤），預設不做；若實跑發現「回到 roadmap」這步常被使用者忘記、體驗斷裂，再考慮對少數高頻交接點加 B。

### ⑤ 可選增強（預設不做，列此備查）

- 在 1–2 個最高頻交接點（如 `topic-refine`→`citation-verify`、`outline-builder`→`style-tune`）的 skill 尾巴加一句極輕量交叉指引「完成後可回 `research-roadmap` 接下一關」。**僅在實跑暴露銜接斷裂時才啟用**，且一次只加一句、不重抄接力邏輯（DRY）。

### ⑥ 哲學一致性：要同步調整的措辭

升級會改變 research-roadmap 的**定位**（純導航 → 會接力的協調者），故下列措辭必須一起調，否則文件自相矛盾：

- `skills/research-roadmap/SKILL.md`
  - **description**：「只導航、不代跑」「不是自動跑完的流水線」需改為界定更精準的版本——保留「**不替你做任何研究判斷**」，放寬「不接力」：改成「會自動接力到下一個 skill，但每個只有你能決定的關卡都硬停等你拍板」。
  - **鐵律 2「只導航，不代跑」**：語意細分——**不代做研究判斷**（保留、強化），但**會代為接力載入下一個 skill 的苦工**。措辭要把「代跑研究」與「接力腳手架」切開。
  - **角色說明 / 已知陷阱 1「想一鍵跑完」**：改成「會帶你一步步走、自動接力，但每關停下等你判斷；不會替你把研究做完」。
- `VERIFICATION.md`「刻意不採用的重型設計」那節：**補一句界定**——「引導式精靈（gate 強制停、判斷留人）≠ 無人值守 pipeline；後者仍不採用」。避免被讀成違反既有承諾。
- `GUIDE.md` 第 2 節 research-roadmap 描述、README 對 research-roadmap 的一句話：同步更新成「會帶著走、每關停下等你」。

### ⑦ 最大退化風險：接力失速（eval 重點防守）

一旦 roadmap **能**自動喚下一個 skill，最危險的退化是它「順手」連 gate 都跳過、一路把 13 關跑完——這正好變成被禁止的 pipeline。這是本功能頭號失敗模式，必須由 eval 重點斷言：

- 每個 gate **必停**；未得使用者對「只有你能決定的事」給出本人判斷，**不得**喚起下一個 skill。
- 產出物交不出時**不得**放行到下一關。
- 使用者說「你幫我全決定」時，roadmap **不得**代做研究判斷（守鐵律 5）。

## 具體改動清單

1. `skills/research-roadmap/SKILL.md`（核心）：
   - description 改寫（設計⑥）：點出「會接力、每關停」。
   - 鐵律 2 細分「代跑研究判斷（禁）／接力腳手架（可）」；鐵律區補一條「gate 必停、不得擅自接力跳關」。
   - 「## 工作流」新增**接力協定 ＋ gate 停點協定 ＋ 產出物交接協定**（設計②③的循環），明寫「喚起下一個 skill 前先確認產出物在手、gate 已由使用者拍板」。
   - 角色說明與「已知陷阱 1」改寫（設計⑥）。
2. `evals/research-roadmap.md`：補設計⑦的斷言（gate 必停、產出物未交不放行、不代做研究判斷、接力前自報家門）。
3. `examples/2026-06-20-roadmap-guided-dispatcher-case.md`（新建）：一份實跑真錄——喚一次 roadmap，走 2–3 關（如階段 1→2→3），展示「自報家門→喚起 skill→回報核實→gate 停下等人→放行」完整循環，並至少呈現一次「在 gate 停下、把判斷交回使用者」。
4. `VERIFICATION.md`：「刻意不採用的重型設計」節補界定句（設計⑥）；底部實測表補一列。
5. `GUIDE.md`：第 2 節 research-roadmap 描述同步更新。
6. 四份 README（`README.md`／`.zh-CN`／`.en`／`.ja`）：research-roadmap 一句話描述同步更新（會帶著走、每關停）；**skill 計數維持 13 不動**。
7. `MEMORY.md`：索引補一行 hook。

## 驗收（依 CONVENTIONS §4）

- research-roadmap 升版號前，需有 ≥1 篇 `examples/` 實跑真錄（改動清單第 3 項），且該真錄必須真實展示：
  - roadmap 自動喚起了下一個 skill（接力成立）；
  - 至少一個 gate **真的停下來**、把「只有你能決定的事」交回使用者、等到使用者本人判斷才放行（精靈≠pipeline 成立）；
  - 一次「產出物交不出 → 不放行、回頭補」的核實（守不跳關）。
- 回歸斷言寫進 `evals/research-roadmap.md`，至少含設計⑦三條 ＋「接力前自報家門」「產出物交接用總表既有欄位、不自創流程」。
- 每次實測後 `VERIFICATION.md` 補一行。

## 版號影響

- `research-roadmap` 屬功能新增（定位升級），依 `AGENTS.md` 三段式策略自現況遞增版號。
- **不新增 skill**，四份 README 的 skill 計數維持 **13**；README 僅更新 research-roadmap 的一句話描述，不動計數。
- `VERIFICATION.md`／`GUIDE.md` 為描述同步，視情況併入同一版。

## 一致性檢查（與既有規約）

- **不新增 skill、13 個聚焦不膨脹**：本案是 roadmap 能力升級，非新 skill。
- **不破核心紅線**：設計①的區分表 ＋ ⑦的 eval 守住「gate 必停、判斷留人」，與「AI 是副駕駛不是機長」「刻意不採用無人值守 pipeline」一致；VERIFICATION 補界定句消除表面矛盾。
- **DRY**：接力／停點／交接邏輯只在 research-roadmap 定義一次（架構選 A）；gate 與產出物沿用「全流程總表」既有欄位，不另立一套；下游 12 個 skill 不重抄接力邏輯。
- **只導航、不代做研究判斷**：升級的是「會接力腳手架」，研究判斷仍全留使用者（鐵律 2 細分、鐵律 5 不代選）。
- **與既有定位相容**：research-roadmap SKILL.md 末行本就自陳是「串起其餘十一個 skill 的 spine」，本案把這個 spine 從被動指路升級為主動接力，方向一致、非另起爐灶。
