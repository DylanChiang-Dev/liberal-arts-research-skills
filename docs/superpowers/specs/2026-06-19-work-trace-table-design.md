# 設計：工作痕跡表（work-trace-table）

> 日期：2026-06-19
> 範圍：本 spec 只涵蓋第 #1 個想法（工作痕跡表）。第 #2 個想法（漂移守衛）另出一份 spec，與本案獨立、互不拖累。

## 背景

我們參考了 orchestra-research 的 `Agent-Native-Research-Artifact` 協議，但**只借想法、不抄任何內容，全部自行重寫**。

借來的想法只有一句：**研究產物應該能標清楚「哪些是人確認的、哪些是 AI 推的」，並把引用綁到查證過的來源、把試過又放棄的方向也留下來。**

對方的原始載體（YAML 的 `exploration_tree`、可執行 DAG、machine-executable layers）是給「跑實驗的 ML 研究」用的。Boya 的使用者是**不會寫程式、在寫論文的文組／人文社科研究者**，所以載體必須是**人讀得懂的 markdown 表**，不是機器 DAG。這一點是刻意的取捨，不照搬原結構。

## 問題

Boya 目前**每一步都在做留痕**——`citation-verify` 的「回源」、`self-review` 的「誠信自查」、`ai-disclosure` 的「留痕自證」都已是各自的鐵律——但這些記錄是**散的**，散落在各 skill 的個別輸出裡，最後拼不成一張完整、可交付的「誰做了什麼」的證據。

當指導教授／期刊／學校誠信辦公室問三句話時，使用者拿不出一份統一的東西回答：

1. 哪些是你自己想的、你決定的？哪些是 AI 幫你弄的？
2. 你引的文獻查證過是真的嗎？
3. 中間有沒有試過又放棄的方向？

## 目標

把「AI 做苦工、你做判斷」這句口號變成一張**看得見、可交付的工作痕跡表**：

- 提供一份**標準格式**（單一真相源），讓全庫對「痕跡」的記法一致。
- 讓**已經在做留痕的三個誠信 skill** 在工作的當下，順手把痕跡填進這張表。
- **不新增 skill**、**不做事後重建**、**不碰其餘 10 個 skill 的內臟**。

## 非目標（明確不做）

- 不新增獨立的「生產痕跡表」skill。理由：會與 `ai-disclosure` 高度重疊（違反 CONVENTIONS 的 DRY、破壞「13 個聚焦 skill」紀律）；且事後靠回憶重建痕跡，正是要避免的「美化／漏記」問題。
- 不採用機器可執行的 DAG／YAML 探索樹。Boya 使用者不維護研究 DAG。
- 不替使用者判斷對錯。標記只記「誰做的」這個事實，「對不對」永遠是使用者的事。
- 不強制其餘 10 個 skill 接入（未來可擴充，但不在本案範圍）。

## 設計

### ① 那張表長什麼樣

一張 markdown 表，使用者放在自己論文資料夾旁邊，每做一件事就追加一列。

欄位：

| 內容（一句話） | 標記 | 來源／原因 | 日期 |
|---|---|---|---|
| 晚清翻譯小說反映文化焦慮 | 🧍 你判斷 | 你的論點 | 2026-06-19 |
| 第二章背景段 | 🤖 AI待核 | AI 整理，待你核 | 2026-06-19 |
| 王德威《被壓抑的現代性》 | ✅ 已查證 | DOI 對得上 | 2026-06-19 |
| 某期刊文章 | ❓ 待查 | 還沒驗 | — |
| 中日比較取向 | 🗑 捨棄 | 範圍太大 | 2026-06-18 |

**五個標記（固定詞彙，單一真相源）：**

| 標記 | 意思 |
|---|---|
| 🧍 你判斷 | 使用者自己想的、自己決定的內容（論點、選擇、詮釋）。 |
| 🤖 AI待核 | AI 幫忙整理／起草的內容，**使用者尚未核實**。核實採納後改標 🧍。 |
| ✅ 已查證 | 引用或事實已驗證為真（例：DOI／來源對得上）。 |
| ❓ 待查 | 尚未查證，或查無、存疑（不確定真偽）。 |
| 🗑 捨棄 | 試過又放棄的方向，**必須在「來源／原因」欄寫放棄原因**（輕量版「彎路保留」）。 |

**欄位規則：**
- 「內容」一句話講清楚是什麼，不需長篇。
- 「來源／原因」：✅ 填查證依據；🗑 填放棄原因；其餘可填一句說明或留空。
- 「日期」可選，未填以 `—` 表示。

### ② 放哪、叫什麼

- **`templates/work-trace-table.md`**：新增一張**空白範本表**（含表頭、五個標記的圖例、1～2 列示意），供使用者複製到自己論文資料夾。檔名與內容繁體中文優先。
- **`CONVENTIONS.md`**：新增一小節「工作痕跡表」，把五個標記的定義與用法**定義一次**。往後任何 skill／文件談痕跡都引用這裡，不重複定義。
- 使用者實際的表存在**他自己的論文資料夾**，Boya 只提供範本與規格，**不規定固定路徑**。

### ③ 三個幫手怎麼邊做邊填（不碰其他 10 個）

實作方式：在這三個 skill 的 `SKILL.md` 各加一個**輸出步驟**，指示 agent 在工作當下，按 `CONVENTIONS.md` 的標準格式，往使用者的工作痕跡表**追加列**。不改其餘邏輯。

- **`citation-verify`（查引用）**：每查完一筆引用 → 加一列。真 → ✅ 已查證；查無／存疑 → ❓ 待查；查出是假的／DOI 貼錯 → 🗑 捨棄並在原因欄記明。
- **`self-review`（自我審查）**：AI 提出的修改建議先標 🤖 AI待核；使用者採納後才改標 🧍 你判斷。意見分級（必改／可辯／誤讀）對應到標記與原因欄。
- **`ai-disclosure`（AI 揭露）**：最後盤點時，**直接讀這張工作痕跡表**生成揭露聲明——表即是它的證據底稿。這把兩者接起來，也正是「不需要新增 skill」的關鍵。

### ④ 邊界（與 Boya 立場一致）

- 標記只記「**誰做的**」的事實，**不替使用者判斷對不對**。
- 工作痕跡表是**輔助、不強制**；但這三個誠信 skill 預設會幫忙填。
- **查無就標 ❓，絕不偽造 ✅**（守 `RULES.md` 與誠信類鐵律「查無≠偽造」「絕不編造」）。
- 不收錄任何規避 AIGC 檢測的用法——本表的立場是**透明留痕**，與 `RULES.md` 第 1 條一致。

## 具體改動清單

1. `CONVENTIONS.md`：新增「工作痕跡表」小節（五個標記定義＋用法＋邊界）。
2. `templates/work-trace-table.md`：新增空白範本表。
3. `skills/citation-verify/SKILL.md`：工作流末加「填寫工作痕跡表」輸出步驟。
4. `skills/self-review/SKILL.md`：同上，含 AI待核→你判斷 的轉換規則。
5. `skills/ai-disclosure/SKILL.md`：改為「讀工作痕跡表作為證據底稿」生成揭露。

## 驗收（依 CONVENTIONS 第 4 條）

- 三個被改動的 skill **升版號前**，各需在 `examples/` 有 ≥1 篇實跑真錄，展示痕跡表被正確填寫／讀取。
- 對應回歸斷言寫進各自的 `evals/<skill>.md`（例：查無時必標 ❓ 不可標 ✅；🗑 必附原因）。
- 每次實測後於 `VERIFICATION.md` 補一行。

## 版號影響

- 三個被改動的 skill 屬功能新增，依 `AGENTS.md` 三段式策略各自升 patch／minor（從現況遞增），不動其餘 10 個。
- `CONVENTIONS.md`、`templates/`、四份 README、`MEMORY.md` 中與本案相關的描述同步更新（README 的 skill 計數不變，仍為 13）。

## 一致性檢查（與既有規約）

- **DRY**：標記定義只在 `CONVENTIONS.md` 出現一次。
- **13 個聚焦 skill 不膨脹**：不新增 skill。
- **當下留痕、非事後重建**：在苦工發生處即填，避免美化／漏記。
- **誠信鐵律**：查無標 ❓、不偽造 ✅、透明不遮掩。
