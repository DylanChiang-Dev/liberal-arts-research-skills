# 計劃：真實期刊事實表（SSCI → TSSCI → CSSCI 分階段建置）

> 立項日：2026-06-23。前置條件：誠信鐵律已於同日鬆綁為「不憑記憶生成＋策展真資料附來源＋查證日期可內建」（見 MEMORY 同日條目）。
> 本計劃只建**客觀事實表**，不做選刊推薦引擎；reach／safe／backup 是使用者相對判斷，永遠留空由使用者填。

## 0. 目標與不做什麼

**目標**：在 `knowledge/` 建一張**每筆附來源＋查證日期**的真實期刊事實表，供 venue-fit 對標與選刊橫向比較使用。所有事實由作者（我）**實抓官網作者須知**填入，抓不到留「待補」，絕不憑記憶。

**刻意不做**（守設計邊界）：
- 不複製 awesome-journal-skills 的逐刊 1984-skill 架構（撞「不讓 skill／庫膨脹」）。
- 不烤入 reach／safe／backup 分級（相對使用者論文，非客觀事實）。
- 不做自動推薦／無人值守爬取；每筆人工查證、人在環中。
- 不憑記憶填影響因子、排名、接受率——這些波動大且非作者須知內容，一律不收（要存只存「作者須知頁明列」的事實）。

## 1. 表格 schema（客觀事實欄）

每筆一列，欄位只記官網明寫的：

| 欄位 | 說明 | 缺漏時 |
|---|---|---|
| 刊名 | 全名（＋常用縮寫） | — |
| 索引／語種 | SSCI／TSSCI／CSSCI；主要語言／地區 | 待補 |
| 學科範圍 | 依該刊 aims & scope | 待補 |
| 字數／結構規範 | 篇幅上限、是否結構化摘要、是否 highlights | 待補 |
| 指定引用格式 | APA 第幾版／Chicago／該刊自訂 | 待補 |
| AI 揭露政策 | 是否有生成式 AI 使用指引、要求 | 待補 |
| 作者須知 URL | 事實來源頁 | **無此即不可入表** |
| 查證日期 | 我實抓當天日期 | **必填** |
| 狀態 | ✅已查證／❓部分待補／🔁待複查（過時效窗） | — |

> 時效窗暫定 **6 個月**；超過標 🔁待複查，venue-fit 引用時提示使用者回官網核對。
> reach／safe／backup 不在此表，留在 venues.md 的「方法＋使用者候選表」那半。

## 2. 學科分群（前十名，廣度目標）

每階段內部再按學科分小批，**每批 3–5 本**。十群（人文社科廣譜，依作者場域排序）：

1. 公共行政／公共政策／數位治理　←　**起步批（最貼作者實測案例）**
2. 政治學／國際關係
3. 社會學
4. 傳播／媒體研究
5. 科技與社會（STS）
6. 資訊系統／資訊管理（IS）
7. 教育學
8. 心理學（社會／教育心理）
9. 管理／組織行為
10. 法律／法政

> 候選刊名待我實抓時逐筆確認其**現行**索引與作者須知，本計劃不預先斷言任何刊的事實。

## 3. 分階段

### 階段一：SSCI（英文國際刊，最易抓、先做）

- 順序：學科群 1 → 2 → … → 10，每群一批、每批 3–5 本。
- 每批工作流（見 §4），跑完留一筆批次記錄。
- 完成判準：十群各至少一批入表、欄位齊或明標待補、全部附 URL＋日期。

### 階段二：TSSCI（台灣繁中刊）

- SSCI 流程跑順後啟動。難度升高：多為 PDF 體例、動態頁。
- 同樣分群、每批 3–5 本；繁中用語與台灣引用慣例另記。

### 階段三：CSSCI（大陸簡中刊）

- 最後做。簡中規範、官網結構差異大，抓取最易卡關。
- 抓不到一律待補，不憑記憶補簡中刊規範。

## 4. 每批標準工作流（誠信落地處）

1. 列該群 3–5 本候選刊名（待查證，不斷言事實）。
2. 逐本 web_fetch 官網「Author Guidelines／投稿須知／Submission」頁。
3. **只記頁面明寫的**，填入 schema，附 URL＋查證日期。
4. 抓不到／動態頁／PDF 卡關／登入牆 → 該欄或整筆標「待補」，**不繞道硬抓、不憑記憶**（守內容取得限制）。
5. 標狀態：全齊 ✅；部分缺 ❓；本批完成記一行（抓了幾本、幾本待補、踩什麼坑）。
6. 工作痕跡標記沿用 CONVENTIONS §9（✅已查證／❓待查）。

## 5. 檔案落點與拆檔觸發

- 起步**長在 `knowledge/venues.md`**（已含 schema 雛形）。
- 觸發拆檔：venues.md 因事實表過長而難讀（≥ ~200 行，或單階段刊數多）→ 拆成 `knowledge/venues-ssci.md`／`venues-tssci.md`／`venues-cssci.md`，venues.md 保留「分級方法」本體並指向。

## 6. 驗證與留痕

- 每階段收尾：抽查數筆對活頁核對；確認**每筆都有 URL＋日期、無任何欄位來自記憶**；過時筆標 🔁。
- 每階段（或每數批）在 `examples/` 留一篇真錄：記實際抓了哪些刊、哪些抓不到及原因、流程踩的坑與寫回。
- `VERIFICATION.md` 視情況補列；`MEMORY.md` 每階段記一行。
- 保鮮：表變大後可考慮一個排程任務定期掃 🔁待複查（待表成形後再評估，非現在）。

## 7. 風險

- **抓取不穩**：期刊官網常 JS 動態渲染或 PDF 須知，web_fetch 可能拿不到 → 待補，不猜。
- **廣度 vs 維護**：十群 ×（SSCI/TSSCI/CSSCI）量不小，單人維護靠「每批 3–5 本、可隨時停」控制節奏；本計劃可逐批交付、不必一次做完。
- **定位漂移**：venues.md 由純方法卡變成「方法＋事實表」，定位略寬；以拆檔觸發（§5）控制。
